# Drift

Локальный веб-браузер для S3 и S3-совместимых хранилищ. Интерфейс написан на React, а ключи доступа используются только Node.js-сервером.

## Что уже умеет

- подключать разные endpoint, бакеты и регионы;
- открывать папки и листать большие списки через S3 continuation token;
- менять размер страницы, искать и сортировать объекты на текущей странице;
- загружать несколько файлов, в том числе перетаскиванием;
- скачивать файлы потоково, без загрузки целого объекта в память сервера;
- создавать папки, переименовывать и переносить файлы и папки;
- рекурсивно удалять папки с пакетами до 1000 объектов;
- показывать изображения, PDF, JSON, код и другие текстовые файлы;
- отдавать Range-запросы для PDF и ограничивать текстовый предпросмотр первыми 512 КБ;
- работать с path-style и virtual-hosted-style S3 endpoint.

## Быстрый старт

Нужен Node.js 22 или новее.

```powershell
npm install
Copy-Item .env.example .env
npm run dev
```

После запуска интерфейс доступен на `http://127.0.0.1:5173`. Vite проксирует API на локальный сервер.

Для production-сборки:

```powershell
npm run build
npm start
```

Готовое приложение откроется на `http://127.0.0.1:4173`.

## Локальное подключение из `.env`

Если задать эти переменные, на стартовом экране появится карточка «Локальный бакет»:

| Переменная | Назначение |
| --- | --- |
| `S3_ENDPOINT` | URL S3-совместимого API |
| `S3_BUCKET` | название бакета |
| `S3_REGION` | регион подписи запросов |
| `S3_ACCESS_KEY_ID` | access key |
| `S3_SECRET_ACCESS_KEY` | secret key |
| `S3_FORCE_PATH_STYLE` | `true` для path-style адресации |
| `HOST` | адрес локального сервера, по умолчанию `127.0.0.1` |
| `PORT` | порт локального сервера, по умолчанию `4173` |

Файл `.env` исключён из Git. В `.env.example` должны оставаться только фиктивные значения.

## Модель безопасности

- Secret Key никогда не возвращается из API в браузер.
- Подключения, введённые через форму, хранятся только в памяти Node.js-процесса и исчезают после перезапуска.
- В `sessionStorage` браузера находится только случайный идентификатор подключения, не ключи S3.
- Сервер по умолчанию слушает только `127.0.0.1` и не пишет тела запросов в логи.
- Потенциально активный контент (`HTML`, `SVG` и неизвестные MIME-типы) скачивается как вложение, а не исполняется в странице.

Текущая версия рассчитана на локальное использование одним пользователем. Перед публикацией в сети нужны HTTPS, авторизация пользователей, CSRF-защита и внешний менеджер секретов. Простая смена `HOST` на `0.0.0.0` не делает приложение безопасным для общего доступа.

## Проверки

```powershell
npm run check
npm run check:s3
```

`check:s3` выполняет только read-only `ListObjectsV2` по настройкам из локального `.env` и проверяет оба варианта S3-адресации.

## Важные детали S3

Перенос объекта в S3 реализован как копирование с последующим удалением, поэтому операция не атомарна. Для больших каталогов она может занять время и потребовать разрешения одновременно на `GetObject`, `PutObject`, `ListBucket` и `DeleteObject`.

Следующими логичными дополнениями будут версии объектов, массовые операции с выделением, скачивание папки архивом, direct multipart upload по временным ссылкам и серверная индексация для поиска по всему бакету.
